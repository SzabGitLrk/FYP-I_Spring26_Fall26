KEY CONCEPTS & DEFINITIONS:
- Apache Kafka (Data Streaming): A distributed system for handling real-time data streams as a continuous flow of records. It acts like a message broker that collects and processes user interactions with Netflix's platform, such as clicks or view history.
   - User Activity → Event Collection/Streaming: The initial step where all the streaming events from users are gathered by Kafka into real-time data processing pipelines.
   
- Apache Spark (Data Processing): A fast and general-purpose cluster computing system that processes streamed data for analysis, including user preferences assessment based on their view history or search activity. It is capable of handling large volumes of streaming information with low latency to provide immediate insights into the collected streams.
   - Analytics & Recommendations: The process where Spark analyzes real-time and historical data from Kafka, applying machine learning algorithms for generating personalized content recommendations based on user behavior patterns such as most watched content
