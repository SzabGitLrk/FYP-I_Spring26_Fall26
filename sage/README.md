`# SAGE — Smart Academic Guide & Engine
**Final Year Project | 100% Offline AI Academic Companion**

SAGE is a desktop study assistant that records lectures, transcribes them,
generates summaries and flashcards, and lets you query your own documents —
all using local models. **No cloud APIs, no data leaves your machine.**

---

## ✨ Features

| Feature | What it does |
|---|---|
| 🎙 **Voice Notes** | Record a lecture, get an instant on-device transcript + AI summary |
| 🃏 **Flashcards** | Auto-generate Q&A study cards from any voice note |
| 📄 **Summarizer** | Drop in a `.pptx` or `.docx`, get a structured academic summary |
| 📊 **Dashboard** | Browse and reopen every saved note, summary, and flashcard set |

Everything is generated by local models running on your CPU. SAGE has been
demoed and tested with the WiFi switched off.

## 🚀 Quick Start

### Prerequisites
- **Python 3.10+**
- **[Ollama](https://ollama.com)** installed
- ~5 GB free disk space (models + cache)
- 8 GB RAM minimum (16 GB recommended for Phi-3.5)

### 1. One-shot install (recommended)

**Windows:**
```powershell
cd sage
.\setup_windows.bat
```

**macOS / Linux:**
```bash
cd sage
bash setup_linux_mac.sh
```

The script creates the virtual env, installs pinned dependencies, and pre-caches
the embedding + STT models so SAGE can run fully offline afterwards.

### 2. Pull the language model (one-time)

```powershell
ollama pull phi3.5
```

If you're on a tight-RAM machine you can use a smaller model instead — SAGE
will auto-fall back through this chain if `phi3.5` isn't installed:

```
phi3.5:3.8b  →  phi3.5  →  tinyllama  →  mistral:7b-instruct-q4_0  →  mistral:7b-instruct
```

Override the default via the `OLLAMA_MODEL` environment variable.

### 3. Start Ollama (separate terminal)

```powershell
ollama serve
```

### 4. Run SAGE

```powershell
cd sage
. .\sage_env\Scripts\Activate.ps1     # PowerShell
# or:  sage_env\Scripts\activate.bat   # cmd.exe
# or:  source sage_env/bin/activate    # macOS/Linux

python main.py
```

A splash screen performs an Ollama health check and warm-loads the embedding
model, then the main window appears.

## 🏗 Architecture

| Layer | Component |
|---|---|
| GUI | CustomTkinter (dark mode, 4-tab layout) |
| Speech-to-Text | Moonshine Tiny / Base ONNX (CPU, ~5–10× faster than Whisper) |
| Language Model | **Phi-3.5-mini 3.8B** via Ollama (Q4 quantized, ~2.2 GB) |
| RAG framework | LangChain 0.2 + ChromaDB 0.5 (local persistent store) |
| Embeddings | `sentence-transformers/all-MiniLM-L6-v2` (22 MB, CPU) |
| Document loaders | `docx2txt` for Word, `UnstructuredPowerPointLoader` for PPT |

### RAG pipeline (shared by all features)

```
text ──► RecursiveCharacterTextSplitter
      ──► HuggingFace embeddings (MiniLM-L6)
      ──► ChromaDB (per-feature collection)
      ──► Retriever (k=4 voice / k=5 documents)
      ──► Phi-3.5 via Ollama + SAGE prompt template
      ──► structured output ──► saved to /storage
```

Every feature (`voice_notes`, `flashcards`, `summarizer`) goes through the
same `rag.pipeline.run_rag()` helper — one code path, easy to maintain.

## 🌐 Fully offline — and how to prove it

SAGE is built to run with no network connection. Three things make this work:

1. **All models are local.** Ollama (Phi-3.5) and Moonshine (ONNX) run on
   your CPU. The MiniLM embedding model is downloaded once during install
   and cached at `~/.cache/huggingface/hub/`.

2. **`main.py` forces offline mode** at startup:
   ```python
   os.environ.setdefault("HF_HUB_OFFLINE", "1")
   os.environ.setdefault("TRANSFORMERS_OFFLINE", "1")
   ```
   Without these flags, `huggingface_hub` would do a HEAD request to
   `huggingface.co` on every model load to check for updates. With them
   set, the libraries skip the network entirely and use the local cache.

3. **ChromaDB telemetry is silenced**:
   ```python
   os.environ.setdefault("ANONYMIZED_TELEMETRY", "False")
   os.environ.setdefault("CHROMA_TELEMETRY_ENABLED", "False")
   # plus a monkey-patch on posthog.capture for older Chroma builds
   ```

**Demo test:** turn on airplane mode → run `python main.py` → record a voice
note → generate flashcards → summarize a `.docx`. Everything works
identically to online operation.

## ⚙️ Configuration

All knobs live in [`config.py`](./config.py). The most useful overrides are
exposed as environment variables:

| Env var | Default | Purpose |
|---|---|---|
| `OLLAMA_MODEL` | `phi3.5` | LLM to use via Ollama |
| `OLLAMA_URL` | `http://localhost:11434` | Ollama HTTP endpoint |
| `SAGE_MOONSHINE_MODEL` | `moonshine/tiny` | STT model (`tiny` or `base`) |

Other tunables in `config.py`:
- `CHUNK_SIZE` / `CHUNK_OVERLAP` — RAG splitter
- `RETRIEVER_K` / `RETRIEVER_K_DOC` — chunks retrieved per query
- `OLLAMA_TEMP` — LLM temperature (default `0.3` for deterministic study output)
- `OLLAMA_CTX` — context window (default `2048`, tuned for 8 GB machines)

## 🐛 Troubleshooting

| Problem | Fix |
|---|---|
| `Cannot connect to Ollama` | Run `ollama serve` in a separate terminal |
| `Model not found` | `ollama pull phi3.5` (or use a fallback model — see Quick Start) |
| `model requires more system memory` | Close other apps, or switch to a smaller model: `set OLLAMA_MODEL=tinyllama` |
| Moonshine import error | `pip install useful-moonshine-onnx==20251121` |
| `Failed to resolve 'huggingface.co'` retries | Already handled — `HF_HUB_OFFLINE=1` is set in `main.py`. If you still see it, your Python process started before that line ran (e.g. a script that imports `sentence_transformers` before importing `main`). |
| `Failed to send telemetry event` from Chroma | Already handled — `posthog.capture` is monkey-patched in `main.py`. |
| `Number of requested results N is greater than number of elements in index M` | Already handled — `run_rag()` caps `k` at the actual chunk count. |
| No audio captured | Check OS microphone permissions for Python / your terminal |
| First STT run is slow | Moonshine downloads ONNX files (~95 MB for `tiny`) once, then caches them |
